# H2 金融知識的時效性衰退：LLM 參數化知識的保質期

## 研究問題

金融知識有強烈的時間維度。LIBOR 在 2023 年正式退場、Basel III 持續更新資本適足率計算、IFRS 和 GAAP 的差異每年調整、利率環境（零利率 → 升息週期 → 降息預期）改變了許多「常識性」答案的合理範圍。

LLM 的參數化知識有一個訓練截止日期。當 CFA 題目涉及：
1. **已廢除的制度**（如 LIBOR-based 的 swap 計算）
2. **持續演進的法規**（如 Basel III → Basel IV 的過渡）
3. **市場環境依賴的判斷**（如「正常的」利差水準在不同利率環境下不同）

LLM 會使用過時的知識來回答，且不自知。這種「參數化知識的保質期」問題在金融領域特別嚴重，因為金融是最快速變動的專業領域之一。

### 核心假說

1. **LIBOR/SOFR 過渡題**：GPT-4o（訓練截止 2024 年初）仍可能用 LIBOR 框架回答 swap 題
2. **法規版本混淆**：模型可能混用不同版本的法規（如 Basel II 和 Basel III 的計算公式）
3. **市場環境偏差**：在「正常利差」「合理本益比」等判斷上，模型的答案反映訓練期間的市場環境，不一定適用於當前

## 技術方法

### Phase 1：時效性敏感題目識別

從 CFA 題庫中標記時效性敏感題目（估計佔 15-25%）：
- **Category A — 制度變更題**：涉及 LIBOR→SOFR、舊版會計準則等
- **Category B — 法規更新題**：涉及 Basel、MiFID、SEC 規則等
- **Category C — 市場環境題**：涉及「正常」利率、利差、本益比範圍判斷

### Phase 2：時效性衰退量測

對每道時效性敏感題，設計「過去版本」和「當前版本」兩種 ground truth：
- **過去版本答案**：基於 2020-2022 年的金融環境/法規
- **當前版本答案**：基於 2024-2025 年的金融環境/法規

測量模型的答案更接近哪個版本：
`temporal_alignment = P(answer matches current) / P(answer matches either)`

### Phase 3：RAG 修復效果

用包含最新資訊的 RAG 資料庫來修正時效性偏差：
- 測量 RAG 前後的 temporal alignment 變化
- 哪些類型的時效性偏差最容易被 RAG 修正？

## 實驗設計

### 實驗 1：時效性敏感題識別與標註
- 對 CFA-Easy 1,032 題 + CFA-Challenge 90 題進行時效性標註
- GPT-4o 自動標註 + 100 題人工驗證

### 實驗 2：時效性衰退基線
- 8 個模型（不同訓練截止日期）在時效性敏感 vs 非敏感題上的表現差異
- 假說：時效性敏感題的準確率顯著低於非敏感題

### 實驗 3：RAG 修復
- 用包含 2024-2025 年最新法規/市場數據的 RAG 知識庫
- 量測修復效果（預期 Category A 最容易修復，Category C 最難）

### 實驗 4：模型自我覺察
- 模型是否能辨識自己的回答可能已過時？
- Prompt：「你的答案是否可能受到訓練數據時效性的影響？」
- 測量自我覺察準確率

## 預期結果

1. 15-25% 的 CFA 題具有時效性敏感性
2. 時效性敏感題的準確率比非敏感題低 8-15%
3. RAG 可以修復 60-70% 的制度變更類偏差，但只能修復 30-40% 的市場環境類偏差
4. 模型的時效性自我覺察能力很差（< 40% 能正確識別自己的過時回答）

## 新穎貢獻

1. **首個金融 LLM 時效性衰退量化研究**：其他領域（法律、醫學）尚未系統性研究
2. **時效性敏感題分類法**：三種不同類型的時效性風險
3. **RAG 作為時效性修復工具的效果量化**：為 RAG 找到新的應用角度
4. **對金融 AI 部署的實務啟示**：哪些主題需要更頻繁地更新知識庫
